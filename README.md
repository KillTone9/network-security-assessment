# Network Infrastructure Security Assessment

## Project Overview

This project focused on evaluating the security posture of a simulated enterprise network infrastructure.

The objective was to identify vulnerabilities, assess associated risks, and propose security controls to improve the overall security of the environment.

---

## Objectives

- Analyze network architecture.
- Identify security weaknesses.
- Evaluate risk levels.
- Recommend mitigation strategies.
- Improve network segmentation and access control.

---

## Environment Analyzed

The assessment was performed on a simulated corporate network consisting of:

- Workstations
- Servers
- Network Switches
- Routers
- Internet Gateway
- Internal Services

---

## Security Assessment Process

### 1. Infrastructure Review

Reviewed:

- Network topology
- Device placement
- Access paths
- User connectivity

---

### 2. Vulnerability Identification

The following weaknesses were identified:

| Vulnerability | Impact |
|-------------|---------|
| Flat network architecture | Lateral movement |
| Weak access controls | Unauthorized access |
| Lack of traffic monitoring | Threats remain undetected |
| Poor segmentation | Increased attack surface |
| Inadequate firewall rules | Excessive exposure |

---

### 3. Risk Assessment

Each vulnerability was evaluated based on:

- Likelihood
- Impact
- Risk level

---

## Risk Matrix

| Risk | Likelihood | Impact | Level |
|--------|------------|---------|--------|
| Unauthorized Access | High | High | Critical |
| Malware Propagation | High | Medium | High |
| Insider Threat | Medium | High | High |
| Data Exposure | Medium | High | High |
| Service Disruption | Medium | Medium | Medium |

---

## Security Recommendations

### Network Segmentation

Implement VLANs to isolate:

- Users
- Servers
- Administrative Systems
- Guest Devices

---

### Firewall Improvements

Implement:

- Default deny policies
- Traffic filtering
- Access control lists

---

### Monitoring

Deploy:

- IDS/IPS solutions
- Centralized logging
- Continuous traffic analysis

---

### Access Control

Apply:

- Least privilege principle
- Role-based access control (RBAC)
- Multi-factor authentication (MFA)

---

## Skills Demonstrated

- Risk Assessment
- Vulnerability Analysis
- Network Security
- Security Architecture
- Security Recommendations
- Threat Identification

---

## Conclusion

The assessment identified several weaknesses that could increase organizational exposure to cyber threats. Recommended controls focused on reducing attack surfaces, improving visibility, and strengthening access management.
